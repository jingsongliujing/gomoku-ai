# 基于飞书多维表格的AI五子棋策略模拟平台

## 作品概述

本项目是一个创新的AI五子棋策略模拟平台，将**自然语言处理**、**AI博弈算法**与**飞书多维表格**深度结合，实现了"人人皆可调参"的AI策略设计体验。用户无需编程知识，只需用自然语言描述自己的下棋风格，系统即可自动解析、对战、评分，形成可视化的竞技排名。

---

## 一、项目背景与意义

### 1.1 AI博弈的传统困境

五子棋作为经典的二人零和博弈游戏，其AI算法研究已有数十年历史。然而，传统的五子棋AI存在以下痛点：

| 痛点 | 表现 | 影响 |
|------|------|------|
| **技术门槛高** | 需要理解棋型评估、权重参数、搜索算法 | 普通用户无法参与AI策略设计 |
| **调试困难** | 参数调整需要反复修改代码、编译、运行 | 策略迭代效率低下 |
| **缺乏协作** | 每个人独立开发，难以分享和比较 | 知识无法积累和传播 |
| **娱乐性弱** | 纯技术驱动，缺乏社交和竞技元素 | 难以吸引大众参与 |

### 1.2 本项目的创新突破

本项目提出了一种全新的AI策略设计范式：

```
传统模式：代码 → 编译 → 测试 → 调参 → 重复
本项目：  自然语言 → 自动解析 → 自动对战 → 自动排名
```

**核心理念：让AI策略设计从"程序员算法工程师专属"变为"人人可参与"**

## 二、技术创新点

### 2.1 自然语言到AI参数的智能映射

本项目的核心创新在于利用大语言模型（DeepSeek）将自然语言策略转换为结构化的AI参数：

```mermaid
graph LR
    A[自然语言策略] --> B[LLM解析]
    B --> C[结构化参数]
    C --> D[AI引擎]
    
    style A fill:#e1f5fe
    style C fill:#f3e5f5
```

**输入示例：**
```
我是一个激进型选手，优先攻击，喜欢构建活三和活四。
开局喜欢占据中心位置。防守时优先堵对方的活三。
```

**输出结果：**
```json
{
    "style": "aggressive",
    "attack_weights": {
        "live_four": 50000,
        "dead_four": 35000,
        "live_three": 25000,
        "dead_three": 18000
    },
    "attack_bias": 1.5,
    "defense_bias": 0.8,
    "center_preference": 2.0,
    "opening_strategy": "center"
}
```

### 2.2 动态权重AI引擎

传统五子棋AI的权重是硬编码的，本项目实现了**完全动态的权重系统**：

```python
class GobangAI:
    def __init__(self, params):
        # 权重完全由参数控制
        self.attack_weights = params.get("attack_weights")
        self.defense_weights = params.get("defense_weights")
        self.attack_bias = params.get("attack_bias", 1.0)
        self.defense_bias = params.get("defense_bias", 1.0)
        self.center_preference = params.get("center_preference", 1.0)
        self.opening_strategy = params.get("opening_strategy", "center")
```

这意味着：
- 同一套AI引擎，可以产生无数种不同的策略风格
- 策略的差异完全由自然语言描述决定
- 用户可以精确控制AI的每一个行为维度

### 2.3 可视化对战与截图上传

每场对战都会：
1. **实时可视化**：弹出Pygame窗口，展示对战全过程
2. **标注胜负**：高亮显示获胜的五子连线
3. **自动截图**：对战结束自动保存截图
4. **上传附件**：截图自动上传到飞书多维表格

这使得对战结果不再是冷冰冰的数据，而是直观、生动的视觉呈现。

### 2.4 ELO等级分排名系统

采用国际象棋通用的ELO等级分系统：

```
新积分 = 旧积分 + K × (实际得分 - 预期得分)

其中：
- K = 32（调节系数）
- 预期得分 = 1 / (1 + 10^((对手积分 - 我方积分) / 400))
```

这种算法的优势：
- 胜利可以获得积分，失败会扣分
- 战胜强敌获得更多积分，输给弱敌扣更多分
- 积分能够真实反映策略水平

## 五、技术实现亮点

### 5.1 零代码策略设计

用户完全不需要编写代码，只需：
1. 打开飞书多维表格
2. 输入自然语言描述
3. 点击保存

系统自动完成：
- 自然语言解析
- AI参数生成
- 对战执行
- 结果记录
- 排名更新

### 5.2 多人协作机制

```
用户A提交策略 ──┐
用户B提交策略 ──┼──▶ 任务队列 ──▶ 依次处理
用户C提交策略 ──┘
```

- 通过飞书"人员"字段关联提交者
- 自动排队处理，避免冲突
- 每人可以提交多个策略

### 5.3 可视化对战

对战过程通过Pygame实时渲染：
- 黑棋 = 用户策略（先手）
- 白棋 = 默认AI（后手）
- 红点 = 最后一步
- 黄圈 = 获胜连线

### 5.4 完整的错误处理

- LLM解析失败 → 自动降级为关键词匹配
- 飞书API限流 → 自动重试
- 对战异常 → 记录失败状态
- 截图上传失败 → 保存本地路径
