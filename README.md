# 飞书多维表格AI五子棋

> 通过自然语言策略控制AI五子棋，与默认AI机器人对战，在飞书多维表格中查看对战结果和排名

## 项目简介

这是一个创新的AI五子棋游戏系统，用户可以通过**自然语言描述**来定义自己的下棋策略，系统会自动将策略解析为AI参数，然后与内置的默认AI机器人进行对战。对战结果（包括截图）会自动保存到飞书多维表格中，形成排行榜。

### 核心特色

- 🎯 **自然语言策略**：用中文描述你的下棋风格，无需编程知识
- 🤖 **自动对战**：提交策略后自动与默认AI机器人PK
- 📸 **对战截图**：每场对战都会生成可视化截图
- 🏆 **排行榜**：ELO积分系统，公平排名
- 👥 **多人排队**：支持多人同时提交，自动排队处理

## 功能说明

### 1. 策略提交

用户在飞书多维表格的"策略提交"表中提交策略：

| 字段 | 说明 |
|------|------|
| 策略名称 | 给你的策略起个名字 |
| 自然语言策略 | 用中文描述你的下棋风格 |
| 提交者 | 自动记录提交人（人员字段） |
| 提交时间 | 自动记录提交时间 |
| 状态 | 待处理 → 解析中 → 对战中 → 已完成 |
| 解析后参数 | 系统自动填充（LLM解析结果） |

### 2. 自动对战流程

```
用户提交策略 → LLM解析为AI参数 → 与默认AI对战 → 生成截图 → 保存结果 → 更新排名
```

- 用户策略AI执**黑棋先手**
- 默认AI机器人执**白棋后手**
- 对战过程全程可视化

### 3. 对战记录

每场对战都会记录：

| 字段 | 说明 |
|------|------|
| 对战ID | 唯一标识 |
| 策略名称 | 使用的策略 |
| 提交者 | 策略提交者 |
| 对战结果 | 胜利/失败/平局 |
| 总步数 | 对战步数 |
| 对战时间 | 对战发生时间 |
| 棋谱 | 完整棋谱（JSON） |
| 对战截图 | 可视化截图（附件） |

### 4. 排行榜

| 字段 | 说明 |
|------|------|
| 策略名称 | 策略名称 |
| 提交者 | 策略提交者 |
| ELO积分 | 等级分（初始1500） |
| 胜场/负场/平局 | 战绩统计 |
| 胜率 | 胜利百分比 |
| 排名 | 按积分排序 |
| 最佳战绩 | 最高积分记录 |

## 安装配置

### 1. 环境要求

- Python 3.8+
- 网络连接（访问飞书API和DeepSeek API）

### 2. 安装依赖

```bash
cd E:\guangzhu\AI_celue\python-gobang
pip install requests pygame
```

### 3. 飞书应用配置

#### 3.1 创建飞书应用

1. 访问 [飞书开放平台](https://open.feishu.cn/)
2. 创建企业自建应用
3. 获取 App ID 和 App Secret

#### 3.2 配置应用权限

在应用管理后台，添加以下权限：

| 权限 | 说明 |
|------|------|
| `bitable:app` | 多维表格应用权限 |
| `wiki:wiki:readonly` | 知识库只读权限 |
| `drive:drive` | 云空间权限（用于上传截图） |

#### 3.3 发布应用

1. 创建应用版本
2. 申请发布
3. 管理员审批通过

### 4. 配置文件

编辑 `feishu_gobang/config.py`：

```python
# 飞书应用配置
FEISHU_APP_ID = "your_app_id"
FEISHU_APP_SECRET = "your_app_secret"

# 多维表格配置
FEISHU_WIKI_TOKEN = "your_wiki_token"  # 从多维表格链接中获取
FEISHU_APP_TOKEN = "your_app_token"    # 通过API获取或手动填写

# LLM配置
LLM_API_BASE = "https://api.deepseek.com"
LLM_API_KEY = "your_deepseek_api_key"
LLM_MODEL = "deepseek-chat"
```

## 使用方法

### 1. 启动程序

```bash
cd E:\guangzhu\AI_celue\python-gobang

# 标准模式（使用LLM解析策略）
python -m feishu_gobang.main

# 不使用LLM（关键词匹配模式）
python -m feishu_gobang.main --no-llm

# 自定义轮询间隔（默认10秒）
python -m feishu_gobang.main --poll-interval 30
```

### 2. 提交策略

在飞书多维表格的"策略提交"表中：

1. 点击"添加记录"
2. 填写"策略名称"（如：激进小王）
3. 填写"自然语言策略"（见下方示例）
4. 将"状态"设置为"待处理"
5. 保存记录

### 3. 策略示例

#### 激进型策略
```
我是一个激进型选手，以进攻为主。
优先构建活三和活四，遇到死四必须立即进攻。
开局喜欢占据中心位置，不断扩大优势。
防守时只在对方形成活三时才防守。
```

#### 防守型策略
```
我是一个防守型选手，稳扎稳打。
优先防守对方的活三和死四，确保自己不输。
进攻时选择最稳妥的方式，不冒险。
开局选择角落位置，慢慢向中心发展。
```

#### 平衡型策略
```
我是一个平衡型选手，攻守兼备。
遇到活四和死四时优先进攻，其他情况先防守。
活三和死三的权重差不多，根据场上形势灵活调整。
开局选择中心位置，保持灵活性。
```

#### 进阶策略
```
我的策略是：
- 活四权重设置为50000，这是必胜的关键
- 死四权重设置为35000，需要立即处理
- 活三权重设置为25000，是重要的进攻手段
- 防守时优先堵对方的活三
- 进攻倾向设置为1.3，稍微激进一点
- 开局选择中心位置
```

### 4. 查看结果

#### 对战记录表
- 查看每场对战的详细过程
- 包含对战截图、棋谱、结果等信息

#### 排行榜
- 查看所有策略的排名
- 包含ELO积分、胜场、负场、胜率等信息

## 棋型说明

### 进攻棋型

| 棋型 | 图示 | 说明 | 默认分值 |
|------|------|------|----------|
| 活四 | `_XXXX_` | 四子相连，两端为空 | 40000 |
| 死四 | `XXXXO` | 四子相连，一端被堵 | 30000 |
| 活三 | `_XXX_` | 三子相连，两端为空 | 20000 |
| 死三 | `XXXO` | 三子相连，一端被堵 | 15000 |
| 活二 | `_XX_` | 两子相连，两端为空 | 1000 |
| 死二 | `XXO` | 两子相连，一端被堵 | 500 |
| 单子 | `_X_` | 单独一子 | 30 |

### 分值规律

```
活四 (40000) > 死四 (30000) > 活三 (20000) > 死三 (15000) > 活二 (1000) > 死二 (500) > 单子 (30)
```

## 策略参数详解

### style - 策略风格

| 值 | 说明 |
|----|------|
| `aggressive` | 激进型，优先进攻 |
| `balanced` | 平衡型，攻守兼备 |
| `defensive` | 防守型，优先防守 |

### attack_bias / defense_bias - 攻防倾向

| 范围 | 说明 |
|------|------|
| 0.5-0.8 | 非常保守 |
| 0.8-1.0 | 稍微保守 |
| 1.0 | 平衡 |
| 1.0-1.3 | 稍微激进 |
| 1.3-2.0 | 非常激进 |

### center_preference - 中心偏好

| 范围 | 说明 |
|------|------|
| 0.5-1.0 | 不太在意中心 |
| 1.0-1.5 | 稍微偏好中心 |
| 1.5-2.0 | 明显偏好中心 |
| 2.0-3.0 | 强烈偏好中心 |

### opening_strategy - 开局策略

| 值 | 说明 |
|----|------|
| `center` | 中心开局（推荐） |
| `corner` | 角落开局 |
| `random` | 随机开局 |

## 常见问题

### Q1: 程序启动后没有反应？

A: 检查以下配置：
1. 飞书App ID和App Secret是否正确
2. 多维表格Token是否正确
3. 应用权限是否已配置并发布

### Q2: 策略解析失败？

A: 可能原因：
1. DeepSeek API Key无效
2. 网络连接问题
3. 策略描述过于复杂

解决方案：
- 检查API Key是否正确
- 尝试使用 `--no-llm` 模式
- 简化策略描述

### Q3: 如何查看对战截图？

A: 在飞书多维表格的"对战记录"表中，每条记录的"对战截图"字段包含可视化截图。

### Q4: 排名多久更新一次？

A: 每次有新策略完成对战后，排名会自动更新。

### Q5: 支持多人同时提交吗？

A: 支持，系统会自动排队处理，不会冲突。
